# Domain Docs

This repository uses a single-context domain documentation layout.

Before engineering work, read `CONTEXT.md` at the repository root and relevant ADRs under `docs/adr/` when they exist. If either location is absent, proceed without treating that absence as an error.

Use terminology defined by `CONTEXT.md` in issues, plans, tests, and implementation. Surface conflicts with an existing ADR explicitly rather than silently overriding the decision.
